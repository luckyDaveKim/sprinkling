# 카카오페이 사전 과제
 카카오페이 뿌리기 기능 구현에 대한 내용이다.

---

# 핵심 문제해결 전략
1. 객체 지향 설계  
-- 객체에서 불필요한 정보 노출을 최소화하고, 자신의 데이터를 최대한 가공하도록 처리했습니다.  
2. 도메인 주도 설계  
-- 비즈니스 로직을 도메인 Layer로 분리했습니다.  
3. 필수 객체 정의를 통한 관리포인트 단일화  
-- 사용자 식별값(`UserId`), 대화방 식별값(`RoomId`), 돈(`Money`), 줍기 인원(`PickerCount`) 를 새로운 타입으로 정의하여 관리포인트를 단일화 했습니다.  
4. **Transaction** 격리 수준 설정  
-- 읽기 및 쓰기별 Transaction 격리 수준을 다르게 설정하여 리소스 낭비를 줄이며 다수의 인스턴스에서의 동작에 대한 신뢰성을 확보했습니다.
5. 요청(req)/응답(res) 처리 객체 분리  
-- 잦은 수정 요구사항이 발생 될 수 있는 요청/응답 객체를 분리하여, 수정에 대한 영향도를 줄였습니다.  
6. 예외 관리 단일화  
-- 뿌리기 서비스 예외는 `BaseSprinklingException` 클래스를 상속하도록 하고, `ErrorCode` 클래스를 통한 예외 관리를 단일화 했습니다.  
7. 뿌리기 분배 로직의 확장성  
-- 뿌리기 분배 로직은 잦은 변경이 발생 할 것으로 판단하여, 확장성을 고려하여 `DividedWay` 인터페이스를 구현하여 바꾸기 손쉽도록 처리했습니다.  
-- ex) **1/n 분배 로직**을 통한 `Split Equally` 구현 등
